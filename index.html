Project Background

Why Lightweight AI-IDS?

The rapid growth of Internet of Things (IoT) devices and increased network interconnectivity have led to higher network traffic and more complex cyber threats. This has left low-resource environments like edge devices and embedded systems especially vulnerable. While AI-based Intrusion Detection Systems (AI-IDSs) offer high detection accuracy and adaptability, they are often too computationally expensive for such environments, creating significant security gaps. The LAIDS (Lightweight AI-based Intrusion Detection System) project aims to address these challenges by:

  • Proposing and developing a series of lightweight AI-IDS architectures
  • Evaluating their performance in low-resource network environments
  • Contributing a real-time IDS framework that balances strong threat detection with computational efficiency

Aims & goals

Overview

The rapid growth of network interconnectivity has resulted in increased network traffic and cyber threat complexity. This has left low-resource networks such as edge devices and embedded systems especially vulnerable. The LAIDS addresses this problem by producing a lightweight Intrusion Detection System (IDS) for real-world deployment. To do this we proposed, investigated, developed and evaluated a series of lightweight AI-based Intrusion Detection Systems, boasting both strong detection performance and low resource consumption. We evaluate

  • The multi-classification performance of the proposed CNN, CNN-LSTM, CNN-GRU, DBN, PCA-CNN, AE-MLP, and GAN-MLP IDSs.
  • The resource consumption of the IDSs after dynamic and full post training quantisation was applied.
  • The ability of the IDSs to detect unknown (zero-day) attacks.

Summary

Project Proposal

The LAIDS Project aimed to design and evaluate a series of lightweight IDSs to strengthen the security of resource-constrained networks such as IoT and edge computing environments. The project sought to address this gap by proposing a series of lightweight AI-based IDS, specifically deep learning architectures to develop efficient, accurate and effective IDS models capable of operating in resource constrained network environments. We proposed six model architectures:

  • A hybrid Convolutional Neural Network/Gated Recurrent Unit (CNN-GRU) model.
  • A hybrid GAN-MLP Generative Adversarial Network/Multilayer Perceptron (GAN-MLP) model.
  • A 1D CNN model enhanced with Principle Component Analysis (CNN-PCA).
  • A hybrid Autoencoder/Multilayer Perceptron (AE-MLP) model.
  • A hybrid Convolutional Neural Network/Long Short-Term Memory (CNN-LSTM) model.
  • A Deep Belief Network (DBN) model.

General Model Pipeline

Preprocess Data

All models were trained and evaluated on the CICIDS2017 network dataset. Data was cleaned, normalised and ADASYN was employed to address class imbalance.

Train Model

All models were trained as a baseline for the architectures.

Test Model

All models were evaluated on unseen testing data to evaluate their multi-classification performance.

Perform Quantisation

Applied dynamic and full post training quantisation, reducing the models to FP32, FP16 and INT8 precisions, before measuring their resource usage.

Test Generalisation

Retrain the models, using zero-day attack class holdouts. The retrained models are then evaluated on the amount of unseen attack class flagged from these holdouts.
